# Mac 模板编辑器操作效率优化设计

日期：2026-07-10

## 背景

当前 Mac 工作区采用功能栏、模板侧栏、预览区和右侧配置区四栏布局。模板配置区同时承担模板、布局、外观、结构和全部图层属性编辑，且“结构”和“图层”职责重叠。复杂模板会产生很长的递归属性列表，新增结构与调整已有组件都需要频繁切换和滚动。

本设计将模板编辑重构为独立的专注设计模式，使搭建层级、直接操作画布和精确调整属性形成一条连续工作流。

## 已确认决策

- 优先目标是操作效率，搭建模板结构与调整已有组件同等重要。
- 采用专注设计模式。进入编辑后，左侧显示完整图层树，右侧只编辑当前对象。
- 只在用户点击“保存模板”时持久化，不自动保存。
- 有未保存修改时退出，提供“保存并退出 / 放弃修改 / 继续编辑”。
- 画布支持点击选中、拖动、缩放和旋转。
- 允许修改 `Watermark.Shared`，为所有组件增加兼容旧模板的统一变换数据。
- 变换采用“原有自动布局结果 + 自由变换增量”，不把模板全面迁移为绝对坐标。
- 第一阶段只支持单选，不实现多选和批量对齐。

## 目标

- 用户可以在同一工作区中快速新增、排序、选择和调整模板组件。
- 画布、图层树和属性面板保持双向选择同步。
- 所有组件具有一致的偏移、缩放和旋转能力。
- 旧模板在没有新变换数据时保持原有渲染结果。
- 未保存编辑不会直接污染当前模板，保存过程不会损坏原配置。
- 连续拖动或滑动只产生一次撤销记录。

## 非目标

- 第一阶段不实现多选、批量对齐或批量分布。
- 不重做 Windows、Android 或 iOS 的模板编辑界面。
- 不用绝对坐标替代现有容器、对齐、边距和自动排列系统。
- 不在第一阶段实现协同编辑、云端草稿或崩溃恢复。

## 整体工作区

用户从模板卡片或当前模板入口进入专注设计模式。该模式隐藏模板库、素材栏和其他工作模式，保留以下区域：

1. 顶部工具栏：退出、模板名称、未保存状态、撤销、重做、缩放和保存模板。
2. 左侧图层树：完整模板层级、新增入口和结构操作。
3. 中间画布：预览、对象命中、选择、拖动、缩放、旋转和辅助线。
4. 右侧属性面板：当前选中对象的上下文属性。
5. 画布状态区：缩放比例、适合窗口和实际尺寸。

进入时对原模板创建深拷贝工作副本。编辑只修改副本；点击“保存模板”后才校验并覆盖原模板。保存成功后清除未保存状态，但用户继续停留在设计模式。

## 组件边界

Mac 专用编辑界面放在 `Watermark.Razor/Components/Mac/`，拆分为：

- `MacTemplateDesigner`：管理工作副本、保存、退出和未保存状态。
- `MacDesignerToolbar`：保存、撤销、重做和缩放命令。
- `MacLayerTree`：层级显示、新增、排序、重命名和移动。
- `MacCanvasEditor`：画布坐标映射、命中测试、直接变换和辅助线。
- `MacSelectionInspector`：根据对象类型选择属性编辑器。
- `MacEditorState`：当前选择、展开节点、缩放、历史记录、渲染版本和脏状态。

共享层只增加通用变换模型、兼容迁移和渲染支持，不包含 Mac UI 状态。

## 图层树

图层树使用单选模式。每个节点显示展开箭头、类型图标、名称、显示状态、锁定状态和拖拽手柄。

交互规则：

- 单击节点选中对象，并同步画布和属性面板。
- 双击名称原地重命名。
- 拖拽节点可同级排序或移动到其他容器。
- 插入位置通过插入线和缩进预览表达。
- 只有容器可以接收子节点。
- 禁止把节点放入自身或其后代，防止层级循环。
- 新增组件时，优先加入当前选中的容器。根级只允许容器；新增文字、Logo 或分割线但没有可用父容器时，先创建一个默认根容器再加入组件。
- 右键菜单提供复制、重命名、移动到和删除。
- `Command+D` 复制，`Delete` 删除，`Enter` 重命名。
- 删除进入撤销历史。删除包含子组件的容器前明确显示影响数量。

画布选中对象后，图层树自动展开其父级并滚动到对应节点。

## 画布直接编辑

渲染器在设计模式下返回预览图片和对象边界字典。`MacCanvasEditor` 将图像像素坐标映射到 CSS 画布坐标，用边界数据完成命中测试和控制点定位。

选中对象后：

- 拖动对象修改相对父容器的 X/Y 偏移。
- 拖动角点统一修改 `ScaleX / ScaleY`，默认等比例缩放。
- 容器显示边线控制点，用于修改 `WidthPercent / HeightPercent` 并让子组件重新排列。
- 文字和 Logo 第一阶段只显示等比例角点，不提供容易造成形变的单边控制点。
- 分割线显示两个端点，用于调整长度和旋转角度。
- 拖动顶部旋转手柄修改角度。
- `Shift` 启用固定步长吸附。
- `Option` 临时关闭吸附。
- 方向键移动 1 个单位，`Shift + 方向键` 移动 10 个单位。
- `Escape` 取消当前指针操作并恢复操作前状态。
- 双击控制点或点击“重置变换”恢复自动布局位置。

辅助线覆盖画布中心、父容器边缘和相邻对象边缘。锁定对象可以被选中查看，但不能通过画布或属性面板修改。

交互期间立即更新控制框，并对低分辨率预览进行节流渲染；指针释放后生成最终清晰预览。每次渲染携带递增版本号，过期结果不得覆盖较新的编辑状态。

## 统一变换模型

在 `Watermark.Shared` 中新增 `WMTransform`，并由所有 `IWMControl` 持有：

```text
OffsetXPercent = 0
OffsetYPercent = 0
ScaleX = 1
ScaleY = 1
Rotation = 0
```

`IWMControl` 同时新增默认值为 `false` 的 `IsLocked` 编辑属性。锁定状态随模板保存，渲染器忽略该字段，只由编辑器限制修改。

渲染步骤为：先执行现有容器、对齐、边距和排列计算，得到对象基准矩形；再以对象中心为默认基点应用偏移、缩放和旋转。

偏移使用父容器百分比，使模板改变画幅后仍保持相对位置。缺少 `WMTransform` 的旧模板使用默认值，因此渲染结果不变。

保存校验将缩放限制在 `0.05` 至 `20`，偏移限制在父容器尺寸的 `-500%` 至 `500%`，旋转保存前归一化到 `[-180, 180)`。属性面板可以在保持比例和独立 X/Y 缩放之间切换；画布角点默认保持比例。

旧 `WMContainer.Angle` 采用兼容读取：旧模板加载时将其映射到新旋转值；容器旋转修改时同步旧字段，避免已有调用路径出现行为分叉。

设计预览新增独立返回类型，包含图像字节和对象边界字典。普通图片生成入口保持原签名，避免改动其他平台调用方。

## 属性面板

右侧面板不再使用“结构 / 画布 / 布局 / 外观 / 图层”五个固定页签，而是跟随当前选择：

- 未选中对象：模板名称、类型、背景、默认图片、画幅和整体效果。
- 容器：图片内容、排列方式、尺寸、变换、背景和效果。
- 文字：EXIF 内容、字体样式、变换和文字边框。
- Logo：图片来源、自动识别、透明处理和变换。
- 分割线：方向、粗细、颜色、长度和变换。

面板顶部显示对象名称、类型、父级路径，以及显示、锁定、复制和删除操作。属性按“内容、布局、变换、外观”分组，默认展开最常用的两组。

控件规则：

- 有明确范围的属性使用滑块加数值框。
- 坐标、角度等使用步进数值框。
- 四边距支持四边联动或独立编辑。
- 颜色使用色块入口。
- 单个字段支持重置，变换组支持整体重置。
- 连续输入在操作结束后合并为一次历史记录。

## 编辑状态与历史记录

编辑数据流：

```text
原模板
  -> 深拷贝工作副本
  -> 编辑事务
  -> 历史快照
  -> 节流设计预览
  -> 显式保存
  -> 原子覆盖原模板
```

历史记录使用有上限的序列化快照，默认保留最近 50 步。图片字节不写入历史，仅保存模板配置和资源路径。

拖动、旋转和连续滑动使用“开始事务 / 提交事务”边界：开始时记录前置快照，过程中只更新工作副本，结束时生成单条历史记录。取消事务时恢复前置快照。

支持：

- `Command+Z` 撤销。
- `Shift+Command+Z` 重做。
- `Command+S` 保存模板。

未保存状态根据当前历史位置和最后一次保存位置计算。保存后仍可继续撤销；撤销到保存位置时未保存标记自动消失。

## 保存与退出

保存流程：

1. 校验工作副本。
2. 将配置写入同目录临时文件。
3. 确认写入成功后原子替换正式配置。
4. 更新当前模板对象并使模板预览缓存失效。
5. 标记当前历史位置为已保存。

退出时：

- 无未保存修改：直接退出。
- 有未保存修改：显示“保存并退出 / 放弃修改 / 继续编辑”。
- 保存失败：保持设计模式打开，不允许覆盖或退出为已保存状态。

由于采用显式保存，应用异常退出时不恢复未保存内容。

## 校验与错误处理

保存前校验：

- 节点 ID 唯一。
- 层级不存在循环。
- 变换和尺寸数值有限且在允许范围内。
- 资源路径可解析。
- 字体和图片引用具有可用回退策略。

校验失败时不覆盖原模板，自动选择并展开第一个出错图层，同时在属性面板显示具体字段错误。

渲染失败时保留上一张有效预览，并显示非阻塞错误提示。缺失的非关键资源使用占位预览并在保存前给出警告；无法保证配置完整性的错误阻止保存。

## 测试策略

共享层单元测试：

- 默认变换不改变旧模板渲染结果。
- `WMTransform` 序列化和反序列化。
- 嵌套对象偏移、缩放和旋转。
- 旧 `WMContainer.Angle` 迁移与同步。
- 设计预览边界与最终渲染位置一致。

编辑状态测试：

- 深拷贝编辑不会修改原模板。
- 拖动和连续滑动只生成一条历史记录。
- 撤销、重做和保存位置正确更新未保存状态。
- 图层排序、跨容器移动和非法循环拦截。
- 保存、放弃和继续编辑三条退出路径。
- 保存失败不会覆盖原配置。

Mac 集成验证：

- 打开旧模板并确认初始预览一致。
- 新增容器、文字、Logo 和分割线。
- 在图层树中排序、跨容器移动和重命名。
- 在画布中拖动、缩放、旋转、吸附和重置。
- 保存后重新打开，层级、位置和样式一致。
- 切换不同画幅后相对偏移合理。
- 缺图、缺字体和渲染异常不会损坏原模板。
- 使用约 30 个组件的复杂模板进行压力验证。

## 验收标准

- 用户可从进入设计模式开始，连续完成新增文字、调整层级、拖动画布、修改字体和保存模板，不需要返回其他工作模式。
- 图层树、画布和属性面板始终指向同一个选中对象。
- 所有组件支持统一偏移、缩放和旋转；重置后回到原自动布局结果。
- 旧模板在未编辑时保持原有生成结果。
- 未点击保存时原模板文件不发生变化。
- 撤销、重做、未保存退出和原子保存按设计工作。
- 复杂模板编辑期间控制框响应流畅，最终预览不会被过期渲染覆盖。
- 在当前目标 Mac 上使用约 30 个组件的模板时，控制框交互目标为 60 FPS，低分辨率预览目标为 250 毫秒内更新，指针释放后的最终预览目标为 1 秒内完成。
